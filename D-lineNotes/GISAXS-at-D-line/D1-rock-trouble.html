Trouble shooting: Rock line-up

  • check whether you have opened the shutter
  • check whether you put a sample on the sample stage, and whether the sample is place correctly
  • check whether you have a sample with a different thickness than the previous one
    • if so, rerun "new", keep sample name and scan number the same, type in a better estimate for the substrate thickness

if all else fails (bad scans continue, limit errors occur):
  • list "user" and "dial" positions : > wsam
  • I lined up the "dial" positions such that the sample holder is parallel to the beam
  • set the samth position to the dial value : > set samth [DIAL]
  •  move samth to 0 : > mv samth 0
  •  restart the rock macro
    • if other motors seem to be off (samx, samaz), you may want to reset them as well
if there are still problems:
  • if samz shows a too large discrepancy between user and dial positions (>0.2 mm), follow the procedure for samth
  • start value for  samz is the negative sample thickness
  • after resetting samth and samz you may also restart from scratch using new 
  • please make sure that 
    • the x-ray shutters are open and that 
    • the substrate thickness is reasonable before you redo rock - these are the 2 most common errors causing run-away scans !
    • my motor limits are set for a reason, so try the above steps first, before changing limits 
    • take a look into the hutch: the sample holder should be horizontal (water level)
